API reference

Every tool the server exposes. Each entry shows the name, what it does, its parameters (required vs optional), and a canonical example. Naming convention: every tool starts with mikrotik_ followed by a verb (create, list, get, update, remove, plus category-specific verbs).

VLAN — Interface Management

Create, list, inspect, update, and remove tagged 802.1Q interfaces. Parent interfaces can be any physical port or bridge.

5 tools
category 01
create mikrotik_create_vlan_interface

Creates a VLAN interface on a parent port.

namerequiredVLAN interface name
vlan_idrequiredVLAN ID (1–4094)
interfacerequiredParent interface
commentoptionalDescription
disabledoptionalCreate in disabled state
mtuoptionalMTU size
use_service_tagoptional802.1ad service tag
arpoptionalARP mode
arp_timeoutoptionalARP timeout
Examplemikrotik_create_vlan_interface(name="vlan100", vlan_id=100, interface="ether1")
list mikrotik_list_vlan_interfaces

Lists VLAN interfaces, optionally filtered.

name_filteroptionalFilter by name
vlan_id_filteroptionalFilter by VLAN ID
interface_filteroptionalFilter by parent interface
disabled_onlyoptionalShow only disabled interfaces
Examplemikrotik_list_vlan_interfaces(vlan_id_filter=100)
get mikrotik_get_vlan_interface

Returns full details for a specific VLAN interface.

namerequiredVLAN interface name
Examplemikrotik_get_vlan_interface(name="vlan100")
update mikrotik_update_vlan_interface

Updates an existing VLAN interface in place. Pass only the fields you want to change.

namerequiredCurrent name
new_nameoptionalRename to
vlan_idoptionalNew VLAN ID
interfaceoptionalNew parent interface
commentoptionalNew description
disabledoptionalEnable / disable
mtuoptionalNew MTU
arpoptionalARP mode
Examplemikrotik_update_vlan_interface(name="vlan100", comment="Production VLAN")
remove mikrotik_remove_vlan_interface

Removes a VLAN interface. Dependent IP addresses, DHCP servers, and firewall rules that reference the interface will be orphaned — clean them up first.

namerequiredVLAN interface name
Examplemikrotik_remove_vlan_interface(name="vlan100")

Firewall — Filter Rules

Manage filter rules across the input, forward, and output chains. Includes rule ordering, enable/disable, basic firewall scaffolding.

9 tools
category 02
create mikrotik_create_filter_rule

Creates a new firewall filter rule. Most match parameters are optional — combine them to build whatever predicate you need.

chainrequiredChain — input, forward, or output
actionrequiredAction — accept, drop, reject, log, jump, …
src_addressoptionalSource address / CIDR
dst_addressoptionalDestination address / CIDR
src_portoptionalSource port
dst_portoptionalDestination port
protocoloptionaltcp, udp, icmp, …
in_interfaceoptionalInput interface
out_interfaceoptionalOutput interface
connection_stateoptionale.g. "established,related"
src_address_listoptionalNamed address list
tcp_flagsoptionalTCP flag matches
limitoptionalRate limit
commentoptionalDescription
disabledoptionalCreate disabled
logoptionalEnable logging
log_prefixoptionalLog prefix string
place_beforeoptionalInsert before this rule ID
Example — allow SSH from LANmikrotik_create_filter_rule( chain="input", action="accept", protocol="tcp", dst_port="22", src_address="192.168.1.0/24" )
list mikrotik_list_filter_rules

Lists filter rules with optional filters by chain, action, addresses, protocol, or state.

chain_filteroptionalFilter by chain
action_filteroptionalFilter by action
src_address_filteroptionalFilter by source address
dst_address_filteroptionalFilter by destination address
protocol_filteroptionalFilter by protocol
interface_filteroptionalFilter by interface
disabled_onlyoptionalOnly disabled rules
invalid_onlyoptionalOnly invalid rules
dynamic_onlyoptionalOnly dynamic rules
Examplemikrotik_list_filter_rules(chain_filter="input")
get mikrotik_get_filter_rule

Returns the full record for a single rule.

rule_idrequiredRouterOS rule ID (e.g. *1)
Examplemikrotik_get_filter_rule(rule_id="*1")
update mikrotik_update_filter_rule

Updates a rule in place. Accepts every parameter from create_filter_rule as optional.

rule_idrequiredRouterOS rule ID
…optionalAll create_filter_rule parameters
Examplemikrotik_update_filter_rule(rule_id="*1", comment="Updated rule")
move mikrotik_move_filter_rule

Moves a rule to a different position in its chain. Critical for ordered-evaluation correctness — e.g. putting a deny rule above an allow rule.

rule_idrequiredRule ID
destinationrequiredNew position (0 = top)
Examplemikrotik_move_filter_rule(rule_id="*1", destination=0)
toggle mikrotik_enable_filter_rule · mikrotik_disable_filter_rule

Toggle a rule on or off without removing it. Useful for emergency disables during troubleshooting.

rule_idrequiredRule ID
Examplemikrotik_disable_filter_rule(rule_id="*1") mikrotik_enable_filter_rule(rule_id="*1")
remove mikrotik_remove_filter_rule

Permanently removes a rule.

rule_idrequiredRule ID
Examplemikrotik_remove_filter_rule(rule_id="*1")
scaffold mikrotik_create_basic_firewall_setup

Creates a sensible baseline: accept established/related, drop invalid, accept ICMP, drop everything else on input. Useful for fresh routers.

—no paramsIdempotent on repeated calls
Examplemikrotik_create_basic_firewall_setup()

NAT — Translation Rules

Source-NAT (masquerade) for outbound, destination-NAT (port forwarding) for inbound, plus rule ordering and toggle.

8 tools
category 03
create mikrotik_create_nat_rule

Creates a NAT rule on the srcnat or dstnat chain.

chainrequiredsrcnat or dstnat
actionrequiredmasquerade, dst-nat, src-nat, redirect, …
to_addressesoptionalTarget address for dst-nat
to_portsoptionalTarget port(s)
protocol · dst_port · …optionalSame predicate set as firewall rules
Example — port-forward HTTPmikrotik_create_nat_rule( chain="dstnat", action="dst-nat", dst_port="80", protocol="tcp", to_addresses="192.168.100.10", to_ports="80", comment="Web server" )
list · get mikrotik_list_nat_rules · mikrotik_get_nat_rule

Inspect NAT rules — optionally filtered by chain. get returns one rule's full record.

Examplesmikrotik_list_nat_rules(chain_filter="srcnat") mikrotik_get_nat_rule(rule_id="*1")
update · move · toggle mikrotik_update_nat_rule · mikrotik_move_nat_rule · mikrotik_enable_nat_rule · mikrotik_disable_nat_rule

Same edit/order/toggle surface as the firewall counterparts.

remove mikrotik_remove_nat_rule
rule_idrequiredRule ID

DHCP — Pools, Networks, Servers

Three-piece model: a pool defines the address range, a network defines gateway and DNS, and a server binds them to an interface.

7 tools
category 04
create mikrotik_create_dhcp_pool · mikrotik_create_dhcp_network · mikrotik_create_dhcp_server

Build a DHCP service for a subnet in three steps.

Example — Guest VLAN DHCPmikrotik_create_dhcp_pool(name="pool-vlan100", ranges="192.168.100.10-192.168.100.200") mikrotik_create_dhcp_network(network="192.168.100.0/24", gateway="192.168.100.1", dns_servers=["8.8.8.8", "8.8.4.4"]) mikrotik_create_dhcp_server(name="dhcp-vlan100", interface="vlan100", address_pool="pool-vlan100")
list · get mikrotik_list_dhcp_servers · mikrotik_get_dhcp_server

List all DHCP servers or fetch one. Use the matching tools for pools and networks if needed.

remove mikrotik_remove_dhcp_server

Removes a DHCP server. The underlying pool and network records survive.

DNS — Resolution & Static Records

Set upstream servers, manage static A/CNAME records, regex blocklists, and inspect the cache. Test queries with mikrotik_test_dns_query.

9 tools
category 05
createmikrotik_set_dns_servers · mikrotik_add_dns_static · mikrotik_add_dns_regexp

Configure upstream resolvers, add static records (A or CNAME), and define regex-based blocklists.

Examplemikrotik_set_dns_servers(servers=["8.8.8.8", "1.1.1.1"], allow_remote_requests=true) mikrotik_add_dns_static(name="router.local", address="192.168.1.1")
read · testmikrotik_get_dns_settings · mikrotik_list_dns_static · mikrotik_get_dns_cache · mikrotik_test_dns_query

Inspect resolver settings, list static records, dump cache, run a live test query.

update · flushmikrotik_update_dns_static · mikrotik_flush_dns_cache · mikrotik_export_dns_config

Edit a static record, clear the resolver cache, or dump DNS configuration to file.

removemikrotik_remove_dns_static
entry_idrequiredStatic record ID

IP Pool — Named Address Pools

Named ranges used by DHCP, hotspot, PPP, and VPN servers. Pools can be inspected and expanded without disruption.

6 tools
category 07
createmikrotik_create_ip_pool
namerequiredPool name
rangesrequirede.g. "192.168.1.100-192.168.1.200"
list · get · usedmikrotik_list_ip_pools · mikrotik_get_ip_pool · mikrotik_list_ip_pool_used

Logs — System Events

Stream and analyze RouterOS logs by topic, severity, or free-text search. Bulk export to CSV. Useful for diagnosing firewall misfires and login attempts.

8 tools
category 12
readmikrotik_get_logs · mikrotik_get_logs_by_severity · mikrotik_search_logs · mikrotik_get_security_logs · mikrotik_get_log_statistics

Five lenses on the log buffer. Filter by topic and limit, or by severity, or by free-text, or just the security topic, or aggregate stats.

Examplemikrotik_get_logs(topics="firewall", limit=100) mikrotik_search_logs(search_term="login failed", case_sensitive=false)
stream · exportmikrotik_monitor_logs · mikrotik_export_logs

Live tail for a duration, or dump to CSV.

clearmikrotik_clear_logs

Clears the in-memory log buffer. Doesn't affect file-backed logs.

Safe Mode — Atomic Sessions

RouterOS safe-mode wraps a sequence of changes; if the management session drops, changes auto-revert. Highly recommended for destructive operations.

5 tools
category 14
beginmikrotik_begin_safe_mode

Starts a new safe-mode session on the device. Subsequent destructive changes are journaled and will be rolled back unless committed.

commit · rollbackmikrotik_commit_safe_mode · mikrotik_rollback_safe_mode

Persist or revert the current safe-mode session.

readmikrotik_get_safe_mode_status · mikrotik_list_safe_mode_changes

Inspect current session state and queued changes.

Pattern — risky firewall edit under safe modemikrotik_begin_safe_mode() mikrotik_create_filter_rule(chain="input", action="drop", src_address="0.0.0.0/0") # test from another window; if still reachable… mikrotik_commit_safe_mode()
REMEMBER
Each tool returns structured JSON. Models that don't natively support function calling can still invoke them via JSON-mode prompts — the schemas are introspectable through the MCP handshake.
